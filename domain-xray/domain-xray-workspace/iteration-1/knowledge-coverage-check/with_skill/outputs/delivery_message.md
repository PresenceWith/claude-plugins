# Delivery Message

## Key Highlights

데이터 엔지니어링 지식 문서에서 백엔드 개발자가 특히 놀라워할 5가지 핵심 인사이트입니다:

1. **멱등성의 의미가 완전히 다릅니다.** 백엔드에서의 멱등성은 "같은 요청을 여러 번 보내도 결과가 같다"이지만, 데이터 파이프라인에서는 "같은 파이프라인을 같은 날짜로 세 번 돌려도 데이터가 세 배가 되지 않는다"를 의미합니다. 이를 위해 INSERT 대신 파티션 단위 DELETE-INSERT 패턴을 기본으로 사용합니다.

2. **조용한 실패(silent failure)가 에러보다 위험합니다.** 파이프라인이 에러로 멈추는 것은 차라리 낫습니다. 에러 없이 잘못된 데이터를 계속 적재하는 것이 진짜 재앙이며, 전문가는 파이프라인 코드보다 데이터 검증 코드에 더 많은 시간을 투자하기도 합니다.

3. **분석 데이터 모델은 의도적으로 중복을 도입합니다.** 3정규형에 익숙한 백엔드 개발자에게는 직관에 반하지만, 분석 환경에서는 JOIN을 줄이는 것이 쿼리 성능과 사용자 편의에 압도적으로 유리합니다. "분석가가 2번 이상의 JOIN 없이 80%의 질문에 답할 수 있어야 한다"가 모델 성공의 기준입니다.

4. **Grain(팩트 테이블의 한 행이 뜻하는 것)을 혼합하면 매출이 3배로 집계됩니다.** 주문 수준 데이터와 주문-상품 수준 데이터를 한 테이블에 섞으면, 숫자가 "합리적으로" 보이면서도 완전히 틀린 결과가 나옵니다. 이 오류는 검증 없이는 발견이 매우 어렵습니다.

5. **파티셔닝 유무로 쿼리 비용이 1000배 차이납니다.** 5TB 테이블에서 하루치 데이터를 조회할 때, 파티셔닝 없으면 $25, 파티셔닝하면 $0.025입니다. BigQuery에서 이런 쿼리가 하루 수십 번 실행되면 월 수만 달러의 차이가 됩니다.
