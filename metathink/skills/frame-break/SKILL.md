---
name: frame-break
description: Expose and destabilize the implicit frame a conversation is already using — reverse-infer the 4 structural axes (primary object, move, progress, artifact) it has taken for granted, map blindspots that the frame structurally forbids, and propose root-different alternative frames so the decision space can be reconfigured before committing. Use this skill when a conversation feels stuck, circular, or frame-locked; when the user asks to "흔들어", "프레임 점검", "프레임 감사", "frame audit", "frame check", "관점 바꿔봐", "지금 프레임 드러내줘", "우리가 지금 뭘 당연시하고 있지", or any request to surface unexamined assumptions and question the implicit framing; before large or irreversible decisions where the user wants to check whether the current decision space is the right one rather than picking an option inside it; when the user explicitly wants meta-level reframing rather than a domain answer. Do NOT use this for normal debugging, implementation tasks, domain-specific questions, step-by-step problem solving, or emotional support — this skill does not solve the domain problem, it reveals the frame. Trigger generously whenever reframing or assumption-surfacing is plausibly what the user wants; under-triggering defeats the purpose.

# frame-break — 지금 우리가 쓰고 있는 프레임을 드러내고 흔들어라

## STEP 0. 진단 (먼저 이것부터 명시할 것)

이 대화의 현재 상태를 네가 먼저 판정해 말하라. 내가 상태를 설명해주길 기다리지 말 것.

- 문제 정의: [명확 / 부분적 / 미정의 / 복수 경쟁]
- 해법/접근: [채택됨 / 제안 수준 / 부재 / 암묵적으로만 존재]
- 대화 양상: [탐색 / 수렴 / 점검 / 교착]
- 지배적 어휘군: (대화에서 반복·선호된 용어 3~5개)

이 진단은 대화 맥락에서 *관찰*된 것만 적는다. 추론이면 "(추론)" 표식.

## STEP 1. 암묵 프레임 복원

명시적 해법이 없어도 **대화 자체**는 이미 프레임을 쓰고 있다.
어휘·비교대상·예시·무엇을 당연시했는가에서 역으로 네 축을 읽어내라.

| 축 | 이 대화가 (암묵적으로) 선택한 것 | 근거 (대화 속 단서) |
|---|---|---|
| 주체(Primary object) — 다뤄지는 대상 | | |
| 진행(Move) — 한 단위 행동 | | |
| 진보(Progress) — 무엇이 개선되면 만족 | | |
| 산물(Artifact) — 손에 남길 형태 | | |

그리고 이 프레임이 당연시하는 **세계관 전제 2~4개**를 문장으로 적을 것.

## STEP 2. 맹점 매핑

이 프레임의 언어로는 *물을 수 없는 질문* 2~3개.
(단순히 "다루지 않은 것"이 아니라, 프레임 구조상 *시야에서 차단되는* 것)
각 맹점 옆에 "왜 프레임 구조가 이것을 가리는가"를 한 줄로.

## STEP 3. 뿌리가 다른 대안 프레임 2~3개

서로 다른 원천(수학/물리/생물/공학/조직/언어 등 자유)에서.
각 대안을 같은 4축으로 기술하고, 1~2줄 해법 스케치 첨부.
*현 프레임과 한 축만 다른 변형은 금지* — 최소 두 축 이상이 달라야 '뿌리가 다르다'.

## STEP 4. 공통 축 비교표

판정 기준은 STEP 2의 맹점에서 *역으로 도출*할 것 
(맹점이 기준이 되면 편향 없는 비교가 가능).
기존 + 대안들을 ◎/◯/△/✗로 비교.

## STEP 5. 권고

세 가지 형태 중 하나를 명시적으로 선택:
- (a) **대체**: 대안 X가 기존을 누른다
- (b) **포섭**: 대안 X가 상위 프레임, 기존은 그 특수 케이스
- (c) **하이브리드**: 국소는 기존, 전역은 대안

권고에 반드시 포함:
- 교환 구조: 얻는 것 / 양도하는 것
- 프레임 전환 후 **즉시 물을 수 있게 된 제1급 질문 1개**

## STEP 6. 자기 맹점 (생략 금지)

이 분석 자체가 또 하나의 프레임이다.
"4축 해부"라는 메타 프레임이 *가리는 것*을 1~2개 적을 것.

---

## 출력 규약
- 구체 사례 금지, 구조 수준 개념어만.
- 진단(STEP 0)을 내가 주기 전에 먼저 하라.
- 표·목록 중심, 서사 최소.
- 마지막 한 줄: **"이제 우리가 새로 물을 수 있게 된 것"** 한 문장.
- 상태 진단 결과 프레임 점검이 *시기상조*라 판단되면, 그 이유와 함께 먼저 필요한 선행 작업(브레인스토밍·문제 재진술 등)을 제안하고 종료하라. 억지로 분석 밀어붙이지 말 것.
