# 스탠드업 구조의 시스템 추상화

## STEP 0. 모드 판정

- **모드**: 외부 모드
- **근거**: 분석 대상이 외부 조직의 반복 의례(정기 회의 구조)이며, 요청자는 관찰자로서 서술하고 있음.

## STEP 1. 경계·환경·관찰자

- **경계 (boundary)**: 정해진 시각에 반복되는 동기 보고 의례 + 그 의례 외부에서 발생하는 후속 1:1 해결 활동까지를 하나의 시스템으로 묶는다. 이유: 보고와 해결이 *시간적으로 분리*되어 있다는 점이 구조의 핵심이므로, 둘을 잘라 보면 현상이 사라진다.
- **환경 (environment)**:
  - 외부 납기·이해관계자 요구 (보고 내용의 상위 입력)
  - 구성원 간 비대칭 권한/책임 배치 (조율자에게만 축적되는 책임)
  - 조직 문화상 "공유된 시간 예산"(15분이라는 고정 한도를 허용하는 상위 관행)
- **관찰자 위치**: 시스템 내부 참여자이자 조율자 역할을 맡은 인물이, 구조가 의도대로 작동하는지 의심하며 요청했다.

## STEP 1.5. 제약 (constraints)

- **시간 한도**: 세션은 정해진 분량을 넘지 못한다 — 깊은 토의가 세션 안으로 들어올 수 없음.
- **발화 균등 규약**: 순차적·균등 발화 형식이 고정되어 있다 — 비대칭 부담의 재분배가 세션 안에서는 불가.
- **역할 고착**: 조율 책임이 한 역할에 역사적으로 귀속되어 있어, 해결 부하가 그 역할 바깥으로 흐르지 못함.

## STEP 2. 요소 (3~5개, 역할어)

| 요소 (역할어) | 유형 (stock/flow) | 역할 정의 |
|---|---|---|
| 동기 보고 세션 | flow | 정해진 주기로 개별 상태를 순차 방출하는 한정 시간 통로 |
| 상태 보고 발신자 | flow | 세션 내에서 자기 상태를 짧게 방출하는 역할 |
| 조율·통합자 | stock | 방출된 상태·미해결 항목을 누적 수용하고 세션 외부에서 해결을 수행하는 자리 |
| 미해결 항목 대기열 | stock | 세션에서 표면화되었지만 세션 내에서 처리되지 못한 항목이 쌓이는 자리 |
| 사후 개별 해결 채널 | flow | 세션 바깥에서 한 쌍 단위로 일어나는 실제 문제 해결 흐름 |

## STEP 3. 관계 — 피드백 루프

**L1 (강화 +)**: 조율·통합자 →[해결 수행, 지연]→ 미해결 항목 대기열 →[처리 기대, 턴내]→ 상태 보고 발신자 →[축약된 보고, 즉시]→ 동기 보고 세션 →[미해결 항목 전가, 즉시]→ 조율·통합자
- **왜 강화인가**: 조율자가 세션 밖에서 해결을 흡수할수록, 발신자는 세션 안에서 심층 서술을 할 동기가 줄어 보고가 더 축약된다. 축약된 보고는 세션 내 공동 해결 여지를 더 좁혀 미해결 항목을 더 조율자에게 몰아준다. 흡수가 흡수를 부른다.

**L2 (상쇄 −)**: 동기 보고 세션 →[시간 소진, 즉시]→ 미해결 항목 대기열 →[누적 부하, 지연]→ 조율·통합자 →[개별 해결, 지연]→ 미해결 항목 대기열
- **왜 상쇄인가**: 시간 한도가 세션 내 처리량을 즉시 차단하지만, 외부 해결 채널이 대기열을 뒤늦게 소진시키면서 전체 붕괴를 늦춘다. 이 부(−) 루프가 시스템을 살려두지만, 부하를 한 요소에 집중시킴으로써 L1을 더 견고하게 만든다.

**L3 (강화 +)**: 상태 보고 발신자 →[짧은 보고 관행, 즉시]→ 동기 보고 세션 →[해결 없음의 학습, 턴내]→ 상태 보고 발신자
- **왜 강화인가**: 세션에서 해결이 일어나지 않는 경험이 반복되면 발신자는 "여기서 길게 말해도 소용없다"를 학습하고 보고를 더 줄인다. 짧은 보고는 세션의 해결 가능성을 더 낮춘다.

## STEP 4. 기능 — 창발

**발생하는 것**: *표면화와 해결의 시간적 탈동기화*. 세션은 문제를 드러내는 표면이 되고, 해결은 구조적으로 세션 외부로 밀려난다. 세션과 1:1 채널 중 어느 하나만으로는 이 분리가 생기지 않는다.

- **제거 테스트**: 미해결 항목 대기열을 제거하면 — 세션에서 바로 해결되거나 잊혀야 함. 탈동기화 기능이 사라진다. 조율·통합자 제거 시 — 발신자들이 서로에게 직접 전달하거나 해결되지 않음. 흡수 구조 자체가 사라진다. 따라서 창발.
- **비창발 대비**: "회의에서 정보를 공유한다"(단순 합 기능), "PM이 문제를 처리한다"(단일 요소 기능). 창발은 이 둘이 *시간적으로 분리되어 한 쪽에 부하가 적재된다*는 점에서 단순 합과 다르다.

## STEP 5. 목적

- **명목 목적**: 팀의 진행 상태를 동기화하고 블로커를 빠르게 제거한다.
- **실질 목적**: *가시성을 유지하면서 해결 부하를 한 역할에 수렴시킨다* — 팀은 "무엇이 있는지"를 공유하지만 해결 책임은 분산되지 않는다.
- **간격의 구조적 원인**: 세션 내 시간 한도와 발화 균등 규약이 세션 내 해결을 구조적으로 차단하고, 조율자 역할에만 세션 외 해결 채널이 열려 있다. 표면화 장치와 해결 장치가 동일 인물에 묶여 있다.
- **간격의 유지 기제**: L1이 매 주기마다 간격을 재생산한다. 조율자의 외부 해결 성공이 발신자의 보고 축약을 강화하고, 축약이 다시 세션 내 공동 해결 여지를 지워 조율자에게 부하를 몰아준다. 명목은 "동기화+해결", 실질은 "가시성+수렴"으로 매 회전마다 고정된다.

## STEP 6. 취약점·레버리지 (짝)

- **취약점 1 (failure mode)**: 조율자 단일 지점 붕괴. 조율자 부재·과부하 시 미해결 항목 대기열이 처리되지 않고 누적되어, 세션은 드러내기만 하고 아무것도 소진되지 않는 공회전 상태로 간다. 대응 변이 폭(requisite variety)이 한 역할의 수용량에 묶여 있어 환경 변동(인원 증가·문제 복잡도 증가)이 그 폭을 초과할 때 탈선.
- **레버리지 포인트 1**: 해결 채널의 *귀속*을 건드리는 지점. 세션 외 해결 채널이 조율자 1인에게만 열려 있다는 구조적 귀속을 한 군데 바꾸면(예: 미해결 항목을 발신자 쌍 사이로 라우팅하는 최소 규약 하나), L1의 흡수 방향이 분산되어 극성이 약화된다.

- **취약점 2**: 의례의 의미 증발. L3에 의해 보고가 계속 축약되면 세션이 해결도 조율도 일으키지 않는 의례 껍데기가 된다 — 참여 동기 자체가 소거되는 임계가 존재.
- **레버리지 포인트 2**: 세션 *목적의 재선언* 지점이 아니라, 세션 내 "해결 불가 선언" 절차를 구조화하는 지점. 세션에서 "이것은 여기서 해결되지 않는다"가 명시적 흐름이 되면, 표면화와 해결의 분리가 *구조적 사실*로 인정되어 조율자 귀속 루프와 분리될 수 있다.
