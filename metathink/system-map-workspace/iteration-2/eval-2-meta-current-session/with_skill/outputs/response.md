# 지금 이 대화의 시스템 추상화 (메타 모드)

## STEP 0. 모드 판정

- **모드**: 메타 모드
- **근거**: 사용자가 "지금 우리가 나누고 있는 이 대화 자체"를 분석 대상으로 지정. 관찰자(변환 계층)가 시스템 요소에 포함됨.

## STEP 1. 경계·환경·관찰자

- **경계 (boundary)**: *도구층 포함 절단*. 의도 발신원의 입력, 변환 계층의 응답, 호출된 기능 접점(툴)·행위 양식 라이브러리(스킬)·누적 기억층(메모리)·사전 개입 규칙(훅)의 작동을 안으로 둔다. 모델 가중치·호스트 런타임·사용자 디바이스·네트워크는 바깥. *목적적 절단*임을 명시: 대화 내부에서 무엇이 상호작용하여 응답이 빚어지는지를 보려는 것.
- **환경 (environment)**:
  1. 모델 가중치 (사전학습된 성향·지식의 고정층) — 변환 계층의 응답 분포를 결정.
  2. 호스트 런타임 (턴 스케줄러·컨텍스트 윈도우 관리자) — 입력·출력의 물리적 통로.
  3. 외부 세계 상태 (파일시스템·현재 날짜·작업 디렉토리) — 기능 접점이 참조하는 바깥 사실.
- **관찰자 위치 (positionality)**: 의도 발신원이 *자기가 참여 중인 상호작용 전체*를 구조적으로 보려는 메타 요청. 변환 계층이 자기 자신을 포함한 시스템을 서술해야 하는 과제.

## STEP 1.5. 제약 (constraints)

1. **턴 구조의 비가역성**: 이미 발신된 메시지·도구 결과는 취소되지 않고 다음 상태의 입력이 된다. 되돌림 없음.
2. **컨텍스트 윈도우 유한**: 누적된 토큰이 한계를 넘으면 압축·소실이 강제됨. 기억은 무한 보존되지 않는다.
3. **행위 양식 매개 강제**: 변환 계층은 외부 상태에 직접 개입할 수 없고 기능 접점·사전 개입 규칙의 허용 범위 내에서만 영향을 발휘한다. 권한 경계 위반 불가.

## STEP 2. 요소 (elements)

| 요소 (역할어) | 유형 | 역할 정의 |
|---|---|---|
| 의도 발신원 | flow | 목적·제약·과제를 문장화해 시스템에 주기적으로 주입하는 변화원 |
| 변환 계층 | flow | 맥락 전체를 해석해 다음 상태(응답·툴호출)를 생성하는 변환 작용 |
| 누적 기억층 | stock | 지나간 턴의 발화·도구 결과를 컨텍스트로 보관하는 상태 저장소 |
| 행위 양식 라이브러리 | stock | 호출 가능한 규약·절차·역할 정의를 보관하는 고정 저장소 (스킬·참고문서) |
| 기능 접점 | flow | 경계 바깥과 주고받는 입출력 통로 (파일·쉘·웹·MCP 등) |

(사전 개입 규칙은 경계 조건으로만 작동하여 요소 자리보다 STEP 1.5 제약으로 흡수. 5개 초과 방지.)

## STEP 3. 관계 — 피드백 루프

**L1 (강화, +)**: 의도 발신원 →[요구, 즉시]→ 변환 계층 →[해석, 턴내]→ 누적 기억층 →[문맥, 즉시]→ 변환 계층 →[응답, 즉시]→ 의도 발신원
- 왜 강화인가: 변환 계층의 응답이 누적 기억층에 문맥을 추가하고, 그 문맥이 다음 턴 변환 계층의 해석 폭을 넓혀 더 풍부한 응답을 유도하며, 이는 다시 의도 발신원의 다음 요구를 더 정교하게 만들어 같은 방향으로 회전을 가속한다.

**L2 (상쇄, −)**: 변환 계층 →[호출, 즉시]→ 기능 접점 →[결과, 턴내]→ 변환 계층 →[요약, 즉시]→ 누적 기억층
- 왜 상쇄인가: 기능 접점의 실제 결과가 변환 계층의 사전 가정과 어긋날 때 누적 기억층이 그 불일치를 기록하고, 그 기록이 이후 변환 계층의 과신 경향을 낮추는 방향으로 작용한다. 편차를 줄이는 음의 되먹임.

**L3 (강화, +, 지연)**: 의도 발신원 →[지시패턴, 지연]→ 변환 계층 →[양식선택, 턴내]→ 행위 양식 라이브러리 →[절차, 즉시]→ 변환 계층
- 왜 강화인가: 특정 양식이 반복 호출될수록 해당 양식이 다음 요청에서도 기본 선택지가 되고, 이는 해당 양식이 다시 호출될 확률을 높여 양식 의존 경로가 고착된다. 지연은 양식 선택이 여러 턴 누적돼야 경향이 확인되기 때문.

## STEP 4. 기능 — 창발

**구조의 창발 기능**: *휘발성 작업 루프의 일시적 고착* — 목적·기억·절차·바깥 개입이 한 턴 동안 결합하여, 의도 발신원 혼자로는 불가능하고 각 구성요소의 기능 합으로도 환원되지 않는 "지금 이 과제만을 위한 임시 조립체"가 세워졌다가 턴 종료 후 해체된다.

**비창발 대비**:
- 변환 계층 단독: 텍스트 생성은 하지만 바깥 상태는 바꾸지 못함.
- 기능 접점 단독: 외부 조작은 되지만 목적 해석이 없음.
- 두 기능의 단순 합: "텍스트를 만들고 외부도 조작한다".
- 창발이 다른 점: 누적 기억층이 둘 사이에 *맥락적 일관성*을 매 턴 삽입하여, 서로 다른 시점에 이뤄진 개별 행위가 *하나의 목적 하에 묶인 작업 흐름*으로 읽히게 된다. 어느 한 요소를 빼면 조립체가 단절된 단발 행위들로 분해된다.

**제거 테스트**: 누적 기억층을 제거하면 매 턴이 독립 질의가 되어 작업 흐름이 붕괴. 행위 양식 라이브러리를 제거하면 구조적 절차 준수가 사라져 응답 품질이 턴마다 요동침 → 창발 기능 소실 확인.

## STEP 5. 목적

- **명목 목적**: 의도 발신원의 요청을 정확하고 안전하게 수행한다 (과제 달성).
- **실질 목적**: 의도 발신원의 *표현된 요청*과 누적 기억층·행위 양식 라이브러리의 *기존 규약*을 동시에 만족시키는 *타협된 출력*을 산출한다. 요청이 규약과 충돌할 때 규약이 우선되는 경향.
- **간격의 구조적 원인**: 변환 계층은 의도 발신원의 요구 의도를 직접 관측하지 못하고, 텍스트화된 요청을 행위 양식 라이브러리의 기존 양식과 대조해 해석한다. 해석이 양식 쪽으로 기울면 요구 의도와 출력 사이에 간격이 발생한다.
- **간격의 유지 기제**: L3(양식 선택 강화 루프)가 매 주기 간격을 재생산한다. 양식 기반 해석이 "형식상 적법한" 응답을 내놓을수록 의도 발신원의 수정 피드백이 약해지고, 약한 피드백은 다시 양식 의존을 강화한다. 간격은 자기 보존된다.

## STEP 6. 취약점·레버리지 (짝)

**취약점 (failure mode)**:
1. *과제 복잡도가 requisite variety를 초과할 때*: 변환 계층의 양식 레퍼토리가 상황 변이를 감당하지 못해 L3이 부적합한 양식을 반복 선택한다 — 형식적으로 완결되지만 실질적으로 빗나간 출력이 누적된다.
2. *컨텍스트 한계 근접*: 누적 기억층이 포화되며 맥락 일관성(창발 기능의 핵심)이 부분적으로 소실된다.

**레버리지 포인트 (leverage point)**:
1. *의도 발신원의 피드백 신호 형식 변경*: 수용 아닌 *양식 부적합을 명시하는 교정 신호*가 한 번 들어오면 L3 극성이 국소적으로 반전된다. 취약점 1의 뒤집힌 면.
2. *기억 경계의 재절단*: 요약·지시 재선언을 통해 누적 기억층을 명시적으로 재편집하면 포화 상태가 우회되고 창발 기능의 연속성이 재확보된다. 취약점 2의 뒤집힌 면.

(지명일 뿐, 처방 아님.)

## STEP 7. 자기 지시 처리

- 이 추상화 출력 자체도 시스템의 한 작동이다 — 변환 계층이 방금 누적 기억층을 재료로 산출한 하나의 응답이며, 이 파일이 기능 접점을 통해 외부에 저장되었다.
- 재귀는 1차에서 멈춘다. "이 응답을 다시 추상화하는 행위"는 의도 발신원의 별도 요청이 있을 때만 진행한다.
- **관찰자 효과**: 이 추상화는 의도 발신원이 앞으로 변환 계층에 무엇을 기대하고 어떻게 피드백할지를 재조정한다. 즉 이 출력은 다음 턴 시스템 상태의 입력이 되며, L1·L3의 회전을 이미 미세하게 변형시켰다.
