프리랜서 디자이너 SaaS 플랫폼의 핵심 영역 간 관계를 컨셉트맵으로 정리했습니다.

## 구조 요약

플랫폼은 **4개 영역**으로 구성됩니다:

1. **포트폴리오 관리** — 프로젝트 갤러리, 공개 프로필 페이지, 작업물 업로드, 카테고리/태그
2. **클라이언트 소통** — 메시징 시스템, 피드백 수집, 프로젝트 진행 공유, 파일 공유
3. **견적서 발행** — 견적서 생성, 결제 연동, 템플릿 관리, 계약서 관리
4. **통합 플랫폼 코어** — 사용자 인증, 클라이언트 DB, 알림 시스템, 대시보드

## 핵심 관계 패턴

**워크플로우 흐름 (노란색 화살표):**
공개 프로필로 유입 → 클라이언트 소통 → 견적서 발행이라는 자연스러운 흐름이 존재합니다. 완료된 프로젝트는 다시 포트폴리오로 순환하는 선순환 구조입니다.

**의존성 (빨간 점선):**
- 메시징 시스템과 견적서 생성이 모두 클라이언트 DB에 의존
- 파일 공유는 작업물 업로드 인프라를 공유
- 결제 연동은 계약서 관리가 선행되어야 함

**인프라 지원 (회색 선):**
사용자 인증과 클라이언트 DB가 세 영역 모두를 지탱하는 기반 레이어입니다.

## 인사이트

클라이언트 DB가 전체 시스템의 허브 역할을 합니다. MVP 설계 시 이 흐름의 최소 경로(프로필 → 소통 → 견적)를 먼저 구현하는 것이 효과적입니다.

아직 결정이 필요한 사항:
- 기술 스택 선택
- 결제 시스템 (Stripe vs Toss Payments)
- 호스팅/인프라
- MVP 범위 (전체 기능 vs 코어만 먼저)
