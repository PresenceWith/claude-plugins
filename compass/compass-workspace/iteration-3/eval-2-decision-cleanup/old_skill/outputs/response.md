## 미결정 사항 (5건)

| 항목 | 현재 상태 | 긴급도 |
|------|-----------|--------|
| **데이터베이스** | PostgreSQL vs MongoDB 고민 중 | 지금 결정 필요 — 다른 결정들이 이것에 의존 |
| **API 구조** | REST vs GraphQL 미결정 | 지금 결정 필요 — 프론트엔드 작업 방식에 직접 영향 |
| **CI/CD 파이프라인** | 아직 정해야 함 | 배포 전에 필요하지만 개발 시작과는 독립적 |
| **배포 전략** | 논의 시작 전 | CI/CD가 정해진 후에 맞춰서 결정 가능 |
| **모니터링** | 검토 필요 | 배포 전략이 정해진 후에 결정해도 됨 |

## 어디서부터 풀어야 하나

5개의 미결정 사항 중에서 **데이터베이스 선택**이 가장 먼저입니다. API 구조(REST vs GraphQL)도 DB 특성에 따라 방향이 달라질 수 있고, 데이터 모델링 자체가 프로젝트의 기반이 되기 때문이에요.

그 다음은 **API 구조**를 정하면 프론트엔드(React) 쪽 개발을 바로 시작할 수 있는 상태가 됩니다.

나머지 3개(CI/CD, 배포 전략, 모니터링)는 개발이 진행되면서 정해도 되는 항목들이에요. 지금 당장 막히지는 않습니다.

**DB 선택부터 같이 정리해볼까요?** PostgreSQL과 MongoDB 사이에서 고민하는 포인트가 뭔지 알려주시면 판단에 도움이 될 만한 기준을 정리해드릴게요.
