# 프리랜서 디자이너를 위한 통합 SaaS 플랫폼 기획

## 1. 제품 개요

**핵심 가치 제안:** 프리랜서 디자이너가 흩어져 있는 업무 도구(포트폴리오, 메신저, 스프레드시트, 견적서 양식)를 하나의 플랫폼에서 관리하여, "작업"에만 집중할 수 있게 해주는 올인원 비즈니스 도구.

**타겟 사용자 페르소나:**
- 1~5년차 프리랜서 그래픽/UI/UX/브랜딩 디자이너
- 월 2~8건 정도의 프로젝트를 동시에 진행
- Behance, Notion, 카카오톡, 엑셀을 번갈아 쓰며 비효율을 느끼는 사람
- "디자인은 잘하는데, 비즈니스 관리는 어렵다"고 느끼는 사람

## 2. 전체 기능 구조 (Feature Map)

### A. 포트폴리오 관리
| 기능 | 설명 |
|------|------|
| 프로젝트 등록 | 이미지/영상 업로드, 프로젝트 설명, 사용 툴, 작업 기간 등 입력 |
| 포트폴리오 페이지 생성 | 개인 URL(예: yourname.platname.com) 자동 생성 |
| 테마/레이아웃 커스터마이징 | 디자이너 취향에 맞는 다양한 레이아웃 템플릿 제공 |
| 비공개 포트폴리오 | 특정 클라이언트에게만 공유 가능한 비밀 링크 |
| 조회 통계 | 누가 언제 포트폴리오를 봤는지 트래킹 |
| PDF 내보내기 | 포트폴리오를 PDF로 추출하여 오프라인 공유 |

### B. 클라이언트 소통
| 기능 | 설명 |
|------|------|
| 프로젝트별 대화방 | 클라이언트와 프로젝트 단위로 메시지를 주고받는 공간 |
| 파일 공유 및 버전 관리 | 시안을 올리면 자동으로 v1, v2, v3 버전 관리 |
| 피드백 수집 (이미지 위 코멘트) | 시안 이미지 위에 직접 핀을 찍어 코멘트를 남기는 기능 |
| 승인 워크플로우 | 클라이언트가 "승인" 또는 "수정 요청" 버튼을 누르는 구조화된 프로세스 |
| 알림 시스템 | 이메일/앱 내 알림으로 새 메시지, 피드백, 승인 상태 전달 |
| 클라이언트 포털 | 클라이언트 전용 뷰 - 회원가입 없이 링크만으로 접근 가능 |

### C. 견적서 및 계약 관리
| 기능 | 설명 |
|------|------|
| 견적서 작성 | 항목별 금액, 할인, 세금 포함 견적서 생성 |
| 견적서 템플릿 | 자주 쓰는 견적 항목을 템플릿으로 저장 |
| 온라인 서명 | 클라이언트가 견적서를 확인하고 전자 서명으로 승인 |
| 계약서 연동 | 간단한 프리랜서 계약서 템플릿 제공 및 서명 기능 |
| 세금계산서/인보이스 발행 | 견적서를 기반으로 청구서/인보이스 자동 생성 |
| 결제 상태 추적 | 미수금, 입금 완료, 연체 등 상태 관리 |

### D. 프로젝트/비즈니스 대시보드
| 기능 | 설명 |
|------|------|
| 프로젝트 현황 보드 | 진행 중/완료/대기 프로젝트를 칸반 보드로 관리 |
| 수익 대시보드 | 월별/분기별 수익, 미수금, 클라이언트별 매출 시각화 |
| 일정 관리 | 프로젝트 마감일, 미팅 일정을 캘린더로 관리 |
| 클라이언트 CRM | 클라이언트 연락처, 프로젝트 히스토리, 메모 관리 |

## 3. MVP 범위 설정

### MVP 선정 원칙
1. **프리랜서가 가장 "고통"을 느끼는 문제**를 먼저 해결한다
2. 기존 도구(Behance, Notion 등) 대비 **명확한 차별점**이 있어야 한다
3. 기술적 복잡도가 낮으면서 **사용자 가치가 높은** 기능을 우선한다
4. **하나의 플로우**(포트폴리오 보여주기 -> 견적 보내기 -> 피드백 받기)를 완성한다

### MVP에 포함 (Phase 1 - 약 8~12주)

```
[핵심 플로우]
포트폴리오 공유 -> 클라이언트 연결 -> 견적서 발행 -> 피드백 수집
```

**포트폴리오:**
- 프로젝트 등록 (이미지 업로드 + 설명)
- 개인 포트폴리오 페이지 생성 (기본 템플릿 1~2개)
- 비공개 링크 공유

**클라이언트 소통:**
- 프로젝트별 대화방 (텍스트 + 파일 첨부)
- 클라이언트 링크 초대 (회원가입 불필요)
- 이메일 알림 (새 메시지, 상태 변경)

**견적서:**
- 견적서 작성 (항목별 금액 입력)
- 견적서 PDF 내보내기
- 견적서 온라인 공유 및 승인/거절

**대시보드:**
- 프로젝트 상태 관리 (진행 중/완료/대기)
- 기본 클라이언트 목록

### MVP 이후 (Phase 2 - 확장)

| 우선순위 | 기능 | 이유 |
|----------|------|------|
| 높음 | 이미지 위 코멘트 피드백 | 디자이너 핵심 페인 포인트, 차별화 요소 |
| 높음 | 견적서 템플릿 및 계약서 | 반복 업무 줄여주는 효과 큼 |
| 높음 | 파일 버전 관리 | 시안 관리 혼선 해결 |
| 중간 | 수익 대시보드 | 비즈니스 인사이트 제공 |
| 중간 | 포트폴리오 테마 커스터마이징 | 디자이너 만족도 향상 |
| 중간 | 캘린더/일정 관리 | 있으면 좋지만 대체재가 많음 |
| 낮음 | 전자 서명 | 법적 검토 필요, 복잡도 높음 |
| 낮음 | 결제 연동 | PG사 연동 복잡, 초기엔 수동 관리 |
| 낮음 | 세금계산서 발행 | 규제/회계 연동 필요, 후순위 |

## 4. 기술 스택 제안

아직 기술 스택을 정하지 않았다고 하셨으므로, 이 프로젝트 특성에 맞는 추천안을 제시합니다.

### 추천 스택 (소규모 팀 / 빠른 MVP 기준)

| 레이어 | 추천 | 이유 |
|--------|------|------|
| **프론트엔드** | Next.js (App Router) + TypeScript | SSR/SSG로 포트폴리오 페이지 성능 최적화, 풍부한 생태계 |
| **UI 라이브러리** | Tailwind CSS + shadcn/ui | 빠른 UI 개발, 디자이너 도구답게 깔끔한 UI 구현 용이 |
| **백엔드** | Next.js API Routes 또는 별도 Fastify/Express | MVP 단계에서는 Next.js 내장 API로 충분, 확장 시 분리 |
| **데이터베이스** | PostgreSQL (Supabase 또는 Neon) | 관계형 데이터 구조에 적합, Supabase 쓰면 인증/스토리지 통합 |
| **ORM** | Prisma 또는 Drizzle | 타입 안전한 DB 접근, 마이그레이션 관리 |
| **파일 스토리지** | AWS S3 또는 Cloudflare R2 | 이미지/파일 업로드 필수, R2는 비용 효율적 |
| **인증** | NextAuth.js 또는 Supabase Auth | 소셜 로그인, 매직 링크 등 빠르게 구현 |
| **실시간 통신** | Supabase Realtime 또는 Socket.io | 클라이언트 소통 기능에 필요 |
| **이메일** | Resend 또는 AWS SES | 알림 이메일, 견적서 공유 등 |
| **배포** | Vercel | Next.js 최적화 배포, 자동 프리뷰 |
| **PDF 생성** | React-PDF 또는 Puppeteer | 견적서/포트폴리오 PDF 내보내기 |

### 대안 스택 (풀스택 프레임워크 선호 시)

- **Remix + Fly.io** : 서버 사이드 중심 개발 선호 시
- **SvelteKit + PlanetScale** : 더 가벼운 프레임워크 선호 시
- **Rails + Hotwire + Heroku** : 빠른 프로토타이핑, 성숙한 생태계

## 6. 사용자 플로우 (핵심 시나리오)

### 시나리오 1: 신규 클라이언트 확보
```
1. 디자이너가 포트폴리오 페이지를 만든다
2. 포트폴리오 링크를 잠재 클라이언트에게 공유한다
3. 클라이언트가 포트폴리오를 보고 "프로젝트 문의" 버튼을 누른다
4. 디자이너에게 알림이 가고, 대화방이 자동 생성된다
5. 대화를 통해 요구사항을 파악한 후 견적서를 작성해서 보낸다
6. 클라이언트가 견적서를 승인하면 프로젝트가 시작된다
```

### 시나리오 2: 진행 중인 프로젝트 피드백
```
1. 디자이너가 시안을 프로젝트 대화방에 업로드한다
2. 클라이언트가 이메일 알림을 받고 링크를 클릭한다
3. 클라이언트가 시안을 보고 코멘트를 남긴다
4. 디자이너가 수정 후 새 버전을 업로드한다
5. 클라이언트가 최종 "승인" 버튼을 누른다
6. 프로젝트 상태가 자동으로 업데이트된다
```

## 7. 수익 모델 제안

| 플랜 | 가격 (월) | 포함 내용 |
|------|-----------|-----------|
| **Free** | 0원 | 포트폴리오 3개 프로젝트, 월 1건 견적서, 기본 대화방 |
| **Pro** | 15,000~25,000원 | 무제한 포트폴리오, 무제한 견적서, 커스텀 도메인, 파일 버전 관리 |
| **Business** | 45,000~55,000원 | Pro 전체 + 계약서, 수익 대시보드, 우선 지원, 팀 기능 |

**핵심 전략:** Free 플랜으로 포트폴리오 기능을 널리 쓰게 만들고, 클라이언트 소통/견적서 기능의 사용량 제한으로 Pro 전환을 유도한다.

## 8. 경쟁 분석 및 차별화

| 경쟁 서비스 | 강점 | 약점 (우리의 기회) |
|-------------|------|---------------------|
| Behance/Dribbble | 거대한 디자이너 커뮤니티 | 클라이언트 소통, 비즈니스 관리 기능 없음 |
| Notion | 유연한 문서 관리 | 포트폴리오 전용 아님, 클라이언트 경험 부족 |
| HoneyBook/Dubsado | 프리랜서 비즈니스 관리 | 한국 시장 미지원, 디자이너 특화 아님, 영어 중심 |
| 크몽/숨고 | 국내 프리랜서 매칭 | 플랫폼 수수료, 자체 브랜딩 불가, 소통 도구 빈약 |

**우리의 포지셔닝:** "디자이너의 언어"로 만든 비즈니스 도구. 시각적 피드백, 포트폴리오 중심 클라이언트 경험, 한국 프리랜서 환경에 맞춘 견적/계약 기능.

- 비공개 링크 공유 기능

- 견적서 작성 UI
- 견적서 온라인 공유 및 승인/거절 플로우
- 견적서 PDF 내보내기

- 프로젝트 상태 관리 보드

### Week 11~12: 마무리
- 버그 수정 및 QA
- 랜딩 페이지 제작
- 베타 사용자 모집 및 피드백 수집
- 배포 및 모니터링 세팅

## 10. 다음 단계로의 제안

1. **사용자 인터뷰** - 실제 프리랜서 디자이너 5~10명과 인터뷰하여 가장 큰 페인 포인트를 검증하세요. 위 기능 중 "반드시 있어야 하는 것"과 "있으면 좋은 것"을 구분하는 데 도움이 됩니다.

2. **프로토타입** - Figma로 핵심 플로우(포트폴리오 -> 견적서 -> 피드백) 와이어프레임을 만들어서 사용자 반응을 먼저 확인하세요.

3. **기술 스택 확정** - 팀 구성원의 기술 역량과 선호도를 고려하여 위 추천안 중 하나를 선택하세요.

4. **네이밍 및 브랜딩** - 서비스 이름, 도메인, 기본 브랜드 아이덴티티를 정하세요. 디자이너 대상 서비스인 만큼 브랜딩이 특히 중요합니다.

5. **MVP 범위 재확인** - 이 문서를 기반으로 "정말 첫 버전에 필요한 것"만 남기고 나머지는 과감히 Phase 2로 미루세요. MVP는 작을수록 좋습니다.
