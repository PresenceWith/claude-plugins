# 내부 도구 기술 스택 현황 정리

지금까지 나온 내용을 결정 상태별로 정리했습니다. 총 7개 항목 중 3개가 확정, 1개가 논의 중, 3개가 아직 논의 시작 전입니다.

---

## 확정된 사항 (3건)

| 항목 | 결정 내용 | 비고 |
|------|-----------|------|
| 사용자 인증 | OAuth | 팀 합의 완료 |
| 데이터베이스 | PostgreSQL | 방금 결정 |
| 프론트엔드 | React | 기존 합의 |

이 세 가지는 확정이므로 앞으로 흔들리지 않아야 합니다. 특히 DB가 PostgreSQL로 방금 결정되었으니, 이 결정을 전제로 다른 항목들을 논의할 수 있습니다.

## 논의 중 (1건)

- **API 구조**: REST vs GraphQL 사이에서 결정 못 한 상태입니다.
  - 이 결정은 프론트엔드(React)와 DB(PostgreSQL) 양쪽에 영향을 주므로, 다음으로 가장 우선 결정해야 할 항목입니다.
  - React 기준으로 보면 GraphQL(Apollo Client 등)이 DX 면에서 유리할 수 있고, REST는 팀 경험과 도구 생태계가 더 안정적일 수 있습니다.

## 논의 시작 전 (3건)

- **배포 전략**: 아직 논의 자체가 시작되지 않았습니다. (컨테이너 기반? 서버리스? 전통적 VM?)
- **모니터링**: 배포 전략이 결정되면 그에 맞춰 선택할 수 있습니다.
- **CI/CD 파이프라인**: 배포 전략과 밀접하게 연결되어 있어, 배포 전략 결정 후 함께 논의하는 것이 효율적입니다.

---

## 권장 결정 순서

1. **API 구조** (REST vs GraphQL) — 지금 바로 결정 가능. 프론트/백엔드 개발 모두에 영향.
2. **배포 전략** — API 구조 결정 후. 인프라 전체 방향이 여기서 갈립니다.
3. **CI/CD 파이프라인** — 배포 전략과 함께 결정.
4. **모니터링** — 배포 환경이 정해진 후 선택지가 명확해집니다.

상태 대시보드를 `dashboard.html`로 생성했습니다. 세 컬럼(확정 / 논의 중 / 미결)으로 현황을 한눈에 볼 수 있고, 진행률 바도 포함되어 있습니다.
