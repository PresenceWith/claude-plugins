# 피트니스 트래킹 앱 기능 정리

기능이 정말 많죠. 하나씩 정리해 볼게요.

## 나열된 기능 목록

지금까지 말씀하신 기능을 정리하면 이렇습니다:

1. 운동 기록
2. 식단 관리
3. 소셜 기능
4. 챌린지
5. 코칭 AI
6. 웨어러블 연동
7. 결제 시스템
8. 오프라인 모드
9. 다국어 지원

총 9개 영역인데, 솔직히 이걸 한꺼번에 다 만들면 출시 자체가 안 됩니다. 우선순위를 나눠야 합니다.

## 우선순위 분류

### 1단계: MVP (최소 출시 가능 제품)

이것만 있어도 "피트니스 앱"이라고 부를 수 있는 핵심 기능입니다.

| 기능 | 이유 |
|------|------|
| **운동 기록** | 앱의 존재 이유. 이게 없으면 피트니스 앱이 아닙니다. |
| **식단 관리** | 운동과 식단은 한 세트. 기본적인 칼로리 기록만이라도 있어야 합니다. |
| **결제 시스템** | 수익 모델 없이 나머지를 개발할 수 없습니다. 프리미엄 구독 구조를 초기부터 설계해야 합니다. |

예상 개발 기간: 3~4개월

### 2단계: 사용자 유지 (리텐션)

MVP로 사용자를 모은 뒤, 계속 쓰게 만드는 기능입니다.

| 기능 | 이유 |
|------|------|
| **소셜 기능** | 친구 추가, 운동 공유 정도. 혼자 하면 금방 질립니다. |
| **챌린지** | "30일 플랭크 챌린지" 같은 것. 소셜과 결합하면 리텐션 효과가 큽니다. |
| **오프라인 모드** | 헬스장은 지하에 많고, 야외 러닝 중 데이터가 끊길 수 있습니다. 운동 기록만큼은 오프라인에서 동작해야 합니다. |

예상 개발 기간: 2~3개월

### 3단계: 차별화

경쟁 앱과 차이를 만드는 기능입니다.

| 기능 | 이유 |
|------|------|
| **웨어러블 연동** | Apple Watch, Galaxy Watch 등. 사용자 편의성은 높지만 연동 개발 비용이 큽니다. |
| **코칭 AI** | "오늘 컨디션 기반 운동 추천" 같은 것. 데이터가 쌓여야 의미 있으므로 후순위가 맞습니다. |

예상 개발 기간: 3~4개월

### 4단계: 확장

글로벌 진출을 위한 기능입니다.

| 기능 | 이유 |
|------|------|
| **다국어 지원** | 국내 시장에서 검증된 후에 해도 늦지 않습니다. 다만, 코드 구조는 처음부터 다국어를 고려해서 문자열을 하드코딩하지 않도록 하세요. |

예상 개발 기간: 1~2개월 (번역 포함)

## 지금 당장 해야 할 것

코드를 쓰기 전에 먼저 해야 할 일이 있습니다:

1. **핵심 사용자 시나리오 3개 정의하기** — "사용자가 앱을 열고 오늘 운동을 기록하는 흐름", "식단을 사진으로 기록하는 흐름", "구독 결제하는 흐름". 이 3개를 화면 단위로 그려보세요.

2. **기술 스택 결정하기** — React Native vs Flutter vs 네이티브. 오프라인 모드와 웨어러블 연동을 고려하면 이 선택이 나중에 큰 영향을 줍니다.

3. **데이터 모델 설계하기** — 운동 기록, 식단 기록, 사용자 프로필의 데이터 구조를 먼저 잡아야 합니다. 나중에 AI 코칭을 붙이려면 데이터가 잘 구조화되어 있어야 합니다.

## 주의할 점

- **코칭 AI를 1단계에 넣지 마세요.** 매력적이지만, 학습할 데이터가 없는 상태에서 AI를 넣으면 "랜덤 추천기"가 됩니다.
- **소셜 기능의 범위를 명확히 하세요.** "소셜"이라고 하면 피드, 댓글, 좋아요, DM, 그룹까지 끝이 없습니다. 1단계에서는 "친구 추가 + 운동 기록 공유"로 한정하세요.
- **다국어는 구조만 미리 준비하세요.** i18n 라이브러리를 초기에 적용해두고, 실제 번역은 나중에 하는 게 효율적입니다.
- **오프라인 모드는 전체가 아닌 부분만.** 모든 기능을 오프라인으로 만들면 복잡도가 폭발합니다. "운동 기록"과 "식단 기록"만 오프라인으로 지원하고, 소셜이나 챌린지는 온라인 전용으로 두세요.

## 요약

```
1단계 (MVP)      : 운동 기록 + 식단 관리 + 결제         → 3~4개월
2단계 (리텐션)    : 소셜 + 챌린지 + 오프라인 모드          → 2~3개월
3단계 (차별화)    : 웨어러블 연동 + 코칭 AI               → 3~4개월
4단계 (확장)      : 다국어 지원                          → 1~2개월
```

전체 로드맵은 대략 9~13개월 정도로 보면 됩니다. 먼저 1단계 MVP의 화면 흐름부터 구체적으로 잡아보시겠어요?
