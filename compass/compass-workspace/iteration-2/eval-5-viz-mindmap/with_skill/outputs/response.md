# Planning Compass -- 피트니스 트래킹 앱 마인드맵

와, 꽤 야심 찬 프로젝트네요! 기능 범위가 넓으니까 먼저 전체 구조를 마인드맵으로 정리해서 보여드릴게요.

말씀하신 기능들을 크게 **9개 핵심 영역**으로 나눴습니다:

1. **운동 기록** -- 루틴 관리, 세트/반복 트래킹, GPS 러닝
2. **식단 관리** -- 칼로리 계산, 사진 인식, 영양소 분석
3. **소셜 기능** -- 친구 피드, 그룹 운동, 응원/댓글
4. **챌린지** -- 주간/월간 챌린지, 리더보드, 보상 시스템
5. **코칭 AI** -- 운동 추천, 식단 추천, 진행 분석

그리고 이걸 뒷받침하는 **인프라 영역**:

6. **웨어러블 연동** -- Apple Watch, Galaxy Watch, Fitbit/Garmin
7. **결제 시스템** -- 구독, 인앱 구매, 무료 티어
8. **오프라인 모드** -- 로컬 캐싱, 동기화 큐
9. **다국어 지원** -- i18n 프레임워크, RTL 지원

마인드맵에서 주목할 점은 **기능 간 연결 관계**입니다:
- 코칭 AI가 운동 기록과 식단 관리 데이터에 의존
- 챌린지가 소셜 기능 위에서 동작
- 웨어러블 연동이 운동 기록에 실시간 데이터 제공
- 보상 시스템이 결제 시스템과 연결

마인드맵을 `mindmap.html`로 생성했습니다.

---

다음으로 정리가 필요한 핵심 미결정 사항이 5개 있습니다:

- **기술 스택** -- React Native vs Flutter vs Native
- **수익 모델** -- 프리미엄 vs 구독 vs 광고+인앱
- **AI 엔진** -- 자체 개발 vs 외부 API vs 하이브리드
- **MVP 범위** -- 1차 출시에 포함할 기능 범위
- **타겟 시장** -- 국내 우선 vs 글로벌 동시 런칭

이 중에서 가장 먼저 결정해야 할 것은 **MVP 범위**와 **기술 스택**입니다. 이 두 가지가 나머지 모든 결정의 기반이 되거든요. 어떤 것부터 이야기해볼까요?
